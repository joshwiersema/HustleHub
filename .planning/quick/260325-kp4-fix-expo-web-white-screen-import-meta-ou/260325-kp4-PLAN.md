<objective>
Fix the Expo web white screen caused by "Cannot use 'import.meta' outside a module" SyntaxError.

Purpose: The white screen is the #1 blocker for web preview. Root cause analysis found that zustand v5.0.12's
`esm/middleware.mjs` contains `import.meta.env` references. Metro's default config uses only the `browser`
condition for web platform resolution, which causes zustand's `exports` field to fall through to the `import`
condition (ESM with import.meta). Since Metro serves the bundle as a classic `<script>` (not `type="module"`),
the browser rejects `import.meta` with a SyntaxError before any JS executes.

The fix: Add `react-native` to Metro's `unstable_conditionsByPlatform.web` array so zustand (and any other
package with a `react-native` exports condition) resolves to its CJS build which does not use import.meta.
This is semantically correct since the project uses react-native-web to shim all RN APIs on the web platform.

Output: Updated metro.config.js, verified web bundle compiles without import.meta in executable code.
</objective>

Update metro.config.js to add `react-native` to the web platform's condition names in Metro's resolver.

Replace the current metro.config.js content with:

```js
// Learn more https://docs.expo.io/guides/customizing-metro
const { getDefaultConfig } = require('expo/metro-config');

/** @type {import('expo/metro-config').MetroConfig} */
const config = getDefaultConfig(__dirname);

// Fix: Add 'react-native' condition for web platform resolution.
// Without this, packages like zustand that have both CJS (react-native condition)
// and ESM (import condition) builds get resolved to the ESM version on web.
// The ESM build uses import.meta.env which causes a SyntaxError because Metro
// serves the bundle as a classic <script> (not type="module").
// Since this project uses react-native-web, the react-native condition is correct.
config.resolver.unstable_conditionsByPlatform = {
  ...config.resolver.unstable_conditionsByPlatform,
  web: [
    'react-native',
    ...(config.resolver.unstable_conditionsByPlatform?.web ?? []),
  ],
};

module.exports = config;
```

This makes Metro resolve `zustand/middleware` to `./middleware.js` (CJS, no import.meta) instead of `./esm/middleware.mjs` (ESM, has import.meta.env) when bundling for web.

Do NOT use `babel-plugin-transform-import-meta` or create a custom `web/index.html` with `type="module"` -- those are workarounds. The resolver condition fix addresses the root cause.
