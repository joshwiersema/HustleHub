<objective>
Install Zustand v5 and create the 5 domain stores with AsyncStorage persist middleware that replace the existing direct AsyncStorage CRUD layer. This is the foundational state management that every subsequent plan and phase depends on.

Purpose: Replace imperative AsyncStorage calls with declarative Zustand stores. Per locked decision, each domain gets a separate store with its own persist key following the `@hustlehub/[domain]` pattern. The shard-first design prevents the AsyncStorage CursorWindow 4-6MB crash on Android.

Output: 5 typed Zustand stores with persist middleware, barrel export, zustand installed in package.json.
</objective>

**1. src/store/profileStore.ts** -- `useProfileStore`
- Persist key: `@hustlehub/profile`
- State: `profile: UserProfile | null`, `isOnboarded: boolean`
- Actions: `setProfile(profile: UserProfile)` (also sets isOnboarded to true), `updateProfile(updates: Partial<UserProfile>)`, `markOnboarded()`, `reset()`
- Import `UserProfile` from `../types`
- Add hydration detection: the store must expose a way for the root layout to know when AsyncStorage has been read. Use Zustand persist's built-in `onRehydrateStorage` and `hasHydrated()`. Do NOT add custom `_hasHydrated` state -- instead, consumers check `useProfileStore.persist.hasHydrated()` and subscribe via `useProfileStore.persist.onFinishHydration(callback)`.

**2. src/store/clientsStore.ts** -- `useClientsStore`
- Persist key: `@hustlehub/clients`
- State: `clients: Client[]`
- Actions: `addClient(client: Client)`, `updateClient(id: string, updates: Partial<Client>)`, `deleteClient(id: string)`, `getClient(id: string): Client | undefined`, `reset()`
- Import `Client` from `../types`

**3. src/store/jobsStore.ts** -- `useJobsStore`
- Persist key: `@hustlehub/jobs`
- State: `jobs: Job[]`
- Actions: `addJob(job: Job)`, `updateJob(id: string, updates: Partial<Job>)`, `deleteJob(id: string)`, `getJob(id: string): Job | undefined`, `completeJob(id: string)` (sets status to 'completed'), `reset()`
- Import `Job` from `../types`

**4. src/store/paymentsStore.ts** -- `usePaymentsStore`
- Persist key: `@hustlehub/payments`
- State: `payments: Payment[]`
- Actions: `addPayment(payment: Payment)`, `deletePayment(id: string)`, `getPaymentsByClient(clientName: string): Payment[]`, `reset()`
- Import `Payment` from `../types`

**5. src/store/gameStore.ts** -- `useGameStore`
- Persist key: `@hustlehub/game`
- State: `xp: number`, `level: number`, `hustleBucks: number`, `streak: number`, `lastActivityDate: string | null`, `earnedBadges: string[]`
- Actions: `addXP(amount: number)` (also adds hustleBucks at 50% rate per existing pattern in storage.ts), `updateStreak()`, `earnBadge(badgeId: string)`, `reset()`
- Import `LEVELS` from `../types` for level calculation in addXP

**6. src/store/index.ts** -- Barrel exports
- Re-export all 5 store hooks: `export { useProfileStore } from './profileStore';` etc.
- Do NOT delete `storage.ts` yet -- it stays as deprecated reference until Plan 02 removes its imports

Every store must:
- Use `create<StateType>()(persist(...))` syntax (double parens for TypeScript)
- Use `createJSONStorage(() => AsyncStorage)` for the storage adapter
- Have a `reset()` action that returns state to initial values
- Import types from `../types` (not inline definitions)
- Export the hook as a named export (not default)
