<objective>
Refactor the existing onboarding screens to use the Zustand profileStore instead of the deprecated storage.ts module, and fix back button touch targets to meet DSGN-03 compliance.

Purpose: The existing onboarding code is 90% complete from the first-pass build. The only breaking issue is that `setup-business.tsx` imports `saveProfile`/`setOnboardingComplete` from `storage.ts`, which writes to different AsyncStorage keys than the Zustand persist middleware. This causes profile data to be invisible to the root layout's `Stack.Protected` guard, meaning onboarding re-triggers on every app restart. This plan makes the single surgical fix plus the touch target compliance fix.

Output: Working end-to-end onboarding flow (welcome -> pick-hustle -> setup-business -> tabs) that persists correctly via Zustand and never re-triggers after completion.
</objective>
