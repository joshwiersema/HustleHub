**1. Create `src/utils/gamification.ts`** with three exported pure functions:

- `checkBadges(gameState, stats)`: Takes `{ earnedBadges: string[], streak: number }` from gameStore and `{ totalClients: number, completedJobs: number, totalEarnings: number }` from other stores. Returns `string[]` of newly earned badge IDs (not already in earnedBadges). Badge criteria:
  - `first_client`: totalClients >= 1
  - `five_clients`: totalClients >= 5
  - `first_job`: completedJobs >= 1
  - `ten_jobs`: completedJobs >= 10
  - `twenty_jobs`: completedJobs >= 20
  - `first_100`: totalEarnings >= 100
  - `earn_500`: totalEarnings >= 500
  - `earn_1000`: totalEarnings >= 1000
  - `streak_7`: streak >= 7
  - `streak_30`: streak >= 30

- `getBadgeProgress(badge, stats)`: Takes a `Badge` object and `{ clients: number, completedJobs: number, totalEarnings: number, streak: number }`. Returns `{ badgeId: string, current: number, target: number, label: string }`. Use `Math.min(actual, target)` for current. Label format: "3/5 clients", "$75/$100", "5/7 days". Use the switch-case pattern from RESEARCH.md Pattern 3.

- `getXPForLevel(level, totalXP)`: Takes current level number and total XP. Returns `{ xpIntoLevel: number, xpForNextLevel: number, progress: number }`. Find current level threshold from LEVELS, find next level threshold. `xpIntoLevel = totalXP - currentThreshold`. `xpForNextLevel = nextThreshold - currentThreshold`. `progress = xpIntoLevel / xpForNextLevel` clamped 0-1. For max level (10), return progress=1.

- `getTotalEarningsFromJobs(jobs)`: Takes `Job[]`, returns sum of `price` for all jobs with `status === 'completed'`. This is the Phase 4 proxy for total earnings before Phase 5 payment system exists.

**5. Create `src/components/CelebrationProvider.tsx`:**
React Context provider that manages a queue of celebrations. Exports `CelebrationProvider` component and `useCelebration` hook.

```typescript
interface CelebrationContextType {
  showXPToast: (amount: number) => void;
}
```

Internal state:
- `xpToast: { amount: number, visible: boolean }`
- `celebrationQueue: Array<{ type: 'level-up', level: number } | { type: 'badge', badgeId: string }>`
- `currentCelebration: null | { type: 'level-up', level: number } | { type: 'badge', badgeId: string }`

Logic:
- `showXPToast(amount)`: Sets xpToast to `{ amount, visible: true }`. XP toast is non-blocking (shows alongside queued celebrations).
- Subscribe to gameStore in useEffect (per RESEARCH.md Pattern 2 -- the simpler closure-variable approach, NOT subscribeWithSelector):
  - Track `prevLevel` and `prevBadges` via closure variables initialized from `useGameStore.getState()`.
  - On subscribe fire: if `state.level > prevLevel` -> push `{ type: 'level-up', level: state.level }` to queue. If `state.earnedBadges.length > prevBadges.length` -> find new badge IDs, push `{ type: 'badge', badgeId }` for each. Update closure vars.
- Queue processing: when `currentCelebration` is null and queue is non-empty, shift first item from queue into `currentCelebration`.
- On dismiss of current celebration: set `currentCelebration = null`, which triggers processing next in queue.

Renders:
- `XPToast` with current xpToast state.
- `LevelUpModal` when `currentCelebration?.type === 'level-up'`.
- `BadgeUnlockSheet` when `currentCelebration?.type === 'badge'`. Look up badge from BADGES constant by badgeId.
- Children passed through.

Wrap all children: `<CelebrationContext.Provider value={{ showXPToast }}>{children}<XPToast .../><LevelUpModal .../><BadgeUnlockSheet .../></CelebrationContext.Provider>`
